# Note to the reviewer — flattened structure

This zip is the `wp-to-astro` template. The reviewer's upload tool couldn't handle nested folders, so every file has been **flattened to the root**. The original folder path is encoded in the filename with `__` as the separator.

**Examples:**

| Filename in this zip | Original path |
|---|---|
| `CLAUDE.md` | `CLAUDE.md` (root) |
| `src__layouts__Layout.astro` | `src/layouts/Layout.astro` |
| `src__components__primitives__Button.astro` | `src/components/primitives/Button.astro` |
| `.claude__agents__live-diff-auditor.md` | `.claude/agents/live-diff-auditor.md` |
| `functions__api__contact.ts` | `functions/api/contact.ts` |
| `scripts__audit-live-diff.mjs` | `scripts/audit-live-diff.mjs` |
| `public___headers` | `public/_headers` (three underscores: the first two are the path separator, the third is the file's actual leading underscore — Cloudflare Pages convention) |
| `public___redirects` | `public/_redirects` |
| `reference__seo-map.csv` | `reference/seo-map.csv` |
| `dot-gitignore.txt` | `.gitignore` (renamed; many upload tools reject dotfiles) |
| `dot-nvmrc.txt` | `.nvmrc` |

**Excluded from this zip** (intentionally, to keep the review focused):
- `node_modules/` (deps — install with `npm install`)
- `dist/` (build output)
- `.astro/`, `.wrangler/` (tool caches)
- `package-lock.json` (200KB+, autogenerated — review `package.json` instead)
- Binary favicons (`public/favicon.ico`, `public/favicon.svg`)
- `.vscode/` (editor config, not part of the template)
- Past `reports/` audit outputs (only `.gitkeep` would have been there)

## What this template is

A production-tested Astro 6 + Tailwind 4 template for rebuilding WordPress sites as static sites on Cloudflare Pages, with Mailgun for forms. It distills lessons from a real WP→Astro migration (Joe's Vintage Guitars / JVG) plus three prior review passes (Perplexity, GPT, Gemini adversarial).

## Read order for the reviewer

If you're going to read one thing, read **`CLAUDE.md`** — it's the operating playbook. It documents:

- 12 must-follow rules at the top
- The "Source of truth hierarchy"
- Page archetypes (A: Conversion / B: Reference)
- The visual audit + ship checklist
- Form/RSS/asset-URL/JSON-LD migration sections
- The `live-diff-auditor` agent + cross-model review honesty note
- Astro gotchas including widget-CSS scoping
- A decision log including 13+ template-level lessons learned

After CLAUDE.md, the priority order for adversarial review:

1. **`scripts__audit-live-diff.mjs`** — the load-bearing audit script. Scrutinize for missed edge cases, false positives, or bypassable checks. Already had bugs in the previous pass that Gemini caught; want a fresh look.
2. **`functions__api__contact.ts`** — Cloudflare Pages Function for Mailgun. Hardened with honeypot, CORS lock, rate limit, field allowlist, error sanitization. Review for any remaining security gaps.
3. **`.claude__agents__live-diff-auditor.md`** — Claude Code subagent definition (YAML frontmatter + body) for the qualitative visual audit pass.
4. **`src__layouts__Layout.astro`** — site shell with SEO head + JSON-LD slot.
5. **`src__config__site.ts`** — single source of truth pattern.
6. **`src__components__primitives__*.astro`** — Button, SectionHeader, PageHero primitives.
7. **`astro.config.mjs`** + **`package.json`** + **`public___redirects`** + **`public___headers`** — config / deployment surface.
8. **`README.md`**, **`CLAUDE-PROMPT.md`**, **`reference__README.md`**, **`reference__seo-map.csv`**, **`reference__form-map.csv`** — docs and templates.
9. **`src__pages__rss.xml.ts`**, **`scripts__check-hero-luminance.mjs`**, **`scripts__setup-prefix.mjs`** — supporting scripts.

## What to look for

- **Bugs** in the audit script or contact endpoint (must-fix / should-fix / nit severity)
- **Methodology gaps** in CLAUDE.md — what failure modes are not covered?
- **Anything still in prose** that should be enforced by tooling
- **Cross-cutting issues** the previous review passes might have missed:
  - Pagination / archives / taxonomy methodology is acknowledged as a TODO
  - In-script JSON-LD schema validation was deferred in favor of Google's Rich Results Test as a manual step
  - Puppeteer color extraction was deferred (DevTools manual extract was deemed sufficient at current scale)
  - Three formal model-role prompts (builder/reviewer/auditor) was deferred — only one builder prompt exists
  - Prettier was deferred as opt-in

If you find a real bug or gap, please return a numbered punch list with severity tags. Skip praise. Skip restating what files do.
